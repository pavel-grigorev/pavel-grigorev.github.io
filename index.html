Interface-as-Component for Spring (a.k.a. icomponent)

Adds support for @Component annotation on interfaces making it possible to apply spring-data-like approach to all sorts of things. Here is an example of what you can build with this tool: 

@Service
public interface EmailService {
  @Subject("email.subject.confirmation")
  @Template("confirmation")
  void sendConfirmation(@Param("username") String username, @Param("link") String link, @To String email);

  @Subject("email.subject.welcome")
  @Template("welcome")
  void sendWelcome(@Param("user") @To User user);
}

The source code for this example is available on GitHub: icomponent-demo.

JPQL Builder

Dynamic JPQL query builder. Provides type safety and fluent API. Example: 

SelectBuilder builder = JpqlBuilder.selectBuilder();
Company c = builder.from(Company.class); // Company is an entity class

Select select = builder.select(c);
select.where(c.getStatus()).isNot(Status.DELETED).orderBy(c.getName());

System.out.println("Query: " + select.getQueryString());
System.out.println("Params: " + select.getParameters());

In this example, isNot() can only accept objects of type matching the c.getStatus() return type, making sure to throw a compile-time error in case the status field type changes from Enum to String, for instance.

Entity Loader add-on for the CUBA Platform

Adds an injectable EntityLoader bean that supports declarative approach vs manual building of the data manager. Example: 

@Query("select e from cubatest_Document e where lower(e.name) like concat('%', lower(:name), '%')")
@QueryParam(name = "name", supplier = "getNameFilter")
private EntityLoader<Document, UUID> documentLoader;

private String getNameFilter() {
  return nameFilterField.getValue();
}

private List<Document> getDocuments() {
  return documentLoader.list();
}
